  • 旧ドメインや別サービスのLPに飛んでいないか確認。

②.5 旧ドメイン・旧URLからのリダイレクト

  • ブラウザで旧URLを直接叩いてみて、「自動的に新サイトへ飛ぶか」を確認。
    これにより、古いリンクやブックマークから来たユーザーと検索エンジンの評価を、新サイト側に集約できる。

③ 構造化データ（schema.org）の実装

トップページ（ブランド・サイト全体）

  • 例：@type: "Organization" / @type: "WebSite" を <script type="application/ld+json"> で記述。
    name / url / logo / sameAs（Instagram・HotPepper など）を設定。

店舗ページ（ローカルビジネス）

  • 店名・住所・電話番号・営業時間・画像（店舗写真）が含まれている。

FAQ・サービス（メニュー）

  • 料金・所要時間・校則・スタイリングが苦手な人向けの回答などをQ&A形式で整理し、AIがそのまま引用できるようにする。
  • 「どんな人向け・所要時間・料金・仕上がりイメージ・注意点」が分かると、AIの回答素材としてとても強い。

③.5 ページ構造・見出し（H1 / H2 / H3）の確認

  • デザイン上の大きさではなく、HTMLタグとしての <h1> が複数ないかを確認。
    テンプレートやテーマの仕様で、自動的に複数出ていないかもチェック。
  • 例：店舗ページなら「MEN'S HAIR SENSE 渋谷神南｜メンズ専門美容室」のように、
    店名＋エリア＋業態（メンズ美容室 / メンズヘアサロン など）が分かるテキストになっているか。
  • 「ページタイトル（h1） → セクションタイトル（h2） → 小見出し（h3）」の階層が整理されているかを確認。
  • キーワードの詰め込みすぎは避けつつ、「何のページか」が見出しだけで伝わる状態を目指す。

③.6 canonical / noindex の整理

  • 例：旧LPやテンプレートのコピーなどがある場合は、
    <link rel="canonical" href="https://mens-sense-shibuya.com/正式URL/"> を記載。
  • 検索やAIの材料にしたくないページは、<meta name="robots" content="noindex,follow"> などで除外。

④ コンテンツ整備（AIが答えを作りやすくする）

  • テキスト「COMING SOON」のまま放置されていないかを確認。
    「トップへ戻る」リンクが旧ドメインや別サイトに飛ばないこともチェック。
  • 例：「渋谷で初めてメンズ美容室に行く人が失敗しないコツ」「社会人1年目・就活生向けの髪型選び」など。
    地名＋悩み系の記事は、GEO的にAIの回答素材として有効。
  • 髪質・職業・校則・年代などの目安を書いておくと、ユーザーもAIも「自分に合うメニュー」を選びやすくなる。
  • 例：「20代会社員・直毛・ナチュラルツイストスパイラルパーマ」など。
    画像もAIが理解できるように情報を付与するイメージ。
  • 特に見出し・冒頭文など、AIが要約に使いやすい部分を優先してチェック。

⑤ 実績・レビュー・信頼の見せ方（No.1・AWARD等の出典も含む）

  • ファーストビュー付近で、「実績のあるサロン」であることが一目で伝わるようにする。
  • 受賞年・部門・順位・媒体名などを整理して記載。
  • 例：「〇〇年 HOT PEPPER Beauty AWARD ◯◯部門 口コミ数 全国1位（※2023年4月〜2024年3月 集計）」など。
    小さな注釈でも良いので、「どのデータに基づくNo.1か」が分かるようにしておく。
    必要に応じて、詳細をまとめた説明ページ（受賞・実績まとめ）へリンクする。
  • 必要に応じて Review / AggregateRating を構造化。
    GoogleやAIが「評価の高いサロン」であることを理解しやすくする。

⑥ 最終確認（検索結果・AI表示の確認）

  • 店名・住所・電話番号・営業時間・写真の印象などが整合しているかを確認。

  • 誤った情報が出てくる場合は、自社サイト側に正しい情報を明示し、構造化・FAQなどで補強する。
